# 精要笔记 UI 规范

生成或润色「精要」类笔记时，请按以下结构书写，以保证层级清晰、便于扫读。

---

## 1. 文档头部（必选）

```markdown
# 《标题》（来源 精要）

| 项目 | 内容 |
|------|------|
| **对应** | 教材/课程 第 X 章 |
| **核心问题** | 用一句话概括本章要回答的问题 |

> **一句话结论**：用一两句话直接回答核心问题，方便快速回忆。
```

- 用 **Markdown 表格** 呈现「对应」「核心问题」，不要用纯文字堆砌。
- **一句话结论** 必须放在 **引用块**（`>`）里，和正文区分开。

---

## 2. 概念定义

- 每个重要概念单独成段或成条。
- **术语** 用粗体，格式：**术语（英文）**：解释。
- 需要时可在段末加「检索关键词」行，用 `、` 分隔。

---

## 3. 核心机制 / 怎么运作

- **必须用标准 Markdown 表格**，表头明确（如：步骤、谁、做什么、结果）。
- 流程可用「步骤 → 步骤」或有序列表，不要用纯文字长段描述流程。

示例：

```markdown
| 步骤 | 主体 | 做什么 | 产生什么结果 |
|:----:|------|--------|--------------|
| 1 | 用户 | 创建钱包 | 生成密钥对 |
| 2 | 钱包 | 生成助记词 | 提供备份与恢复手段 |
| 3 | 用户 | 用私钥签名交易 | 交易上链并记录 |
```

---

## 4. 核心要点 / 为什么重要

- 用 **无序列表**（`-`），每条一句或一小段。
- 关键术语继续用 **粗体**。
- 若「核心要点」与「为什么重要」都写，可用 **三级标题** 分开：`### 核心要点`、`### 为什么重要`。

---

## 5. 常见误区 / 风险提醒

- **统一用「误区 → 正确理解」对照**，便于扫读。
- 格式建议：

```markdown
- **误区**：助记词可以只备份一部分。  
  **正确理解**：少一个词就无法恢复，必须完整备份且离线保存。
```

或短句时用加粗：

```markdown
- **误区**：…… → **正确理解**：……
```

---

## 6. 参考来源

- 用列表，每条：`[序号] 标题 - 路径或链接`。
- 若只有当前文档，可写「本文档」或省略该节。

---

## 7. 通用排版

- 大节之间用 `---` 分隔。
- 小节用 `##`、`###`，层级不超过 3 级。
- 列表与段落之间空一行；表格前后各空一行。
- **检索关键词** 放在对应小节末尾，可统一用一句「**检索关键词**：……」。

按此规范输出的笔记在预览或编辑器中会层次分明、表格与引用块醒目，UI 更清晰。
